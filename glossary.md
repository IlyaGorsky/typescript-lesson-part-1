# Глоссарий TypeScript — Часть 1

| Термин | Описание |
| ------ | -------- |
| **TypeScript** | Надмножество JavaScript с системой статических типов. Код компилируется в JS перед выполнением |
| **Статическая типизация** | Проверка типов на этапе компиляции, а не во время выполнения |
| **Type Annotation** | Явное указание типа переменной через двоеточие: `let name: string` |
| **Type Inference** | Автоматический вывод типа компилятором без явной аннотации: `let x = 5` → тип `number` |
| **Primitive Types** | Базовые типы JS: `number`, `string`, `boolean`, `bigint`, `symbol` |
| **number** | Тип для целых и дробных чисел, `NaN`, `Infinity` |
| **string** | Тип для строк — обычных и шаблонных (template literals) |
| **bigint** | Тип для целых чисел произвольной точности (`100n`) |
| **boolean** | Тип с двумя значениями: `true` и `false` |
| **symbol** | Тип для уникальных идентификаторов, создаётся через `Symbol()` |
| **void** | Тип для функций, которые ничего не возвращают |
| **undefined** | Тип и значение — переменная объявлена, но не инициализирована |
| **null** | Тип и значение — явное отсутствие значения |
| **any** | «Выключает» проверку типов — допускает любые операции. Следует избегать |
| **unknown** | Безопасная альтернатива `any` — значение можно присвоить, но нельзя использовать без проверки типа |
| **never** | Тип, у которого нет значений — функция, которая никогда не завершается (throw, бесконечный цикл) |
| **enum** | Перечисление — набор именованных констант. Бывает числовым и строковым |
| **Object Type** | Описание формы объекта через набор свойств и их типов: `{ name: string; age: number }` |
| **Array** | Типизированный массив: `number[]` или `Array<number>` |
| **Tuple** | Кортеж — массив фиксированной длины с типом для каждой позиции: `[string, number]` |
| **Type Alias** | Псевдоним типа через ключевое слово `type`. Может описывать примитивы, объединения, кортежи, объекты |
| **Interface** | Именованный контракт для описания формы объекта. Поддерживает `extends` и declaration merging |
| **Optional Property** | Необязательное свойство интерфейса/объекта, помечается `?`: `age?: number` |
| **Readonly Property** | Свойство, доступное только для чтения: `readonly id: number` |
| **extends** | Наследование интерфейсов — дочерний интерфейс получает все свойства родительского |
| **Intersection (&)** | Комбинирование типов — результат содержит все свойства обоих типов |
| **Union (\|)** | Объединение типов — значение может быть одним из перечисленных |
| **Literal Type** | Конкретное значение как тип: `"hello"`, `42`, `true` |
| **Template Literal Type** | Шаблонный литеральный тип: `` `hello ${string}` `` — строки по шаблону на уровне типов |
| **Function Type** | Описание типа функции: `(a: number, b: number) => number` |
| **tsconfig.json** | Файл конфигурации TypeScript — параметры компиляции и включаемые файлы |
| **strict** | Флаг в tsconfig, включающий все строгие проверки типов |
| **tsc** | Компилятор TypeScript — проверяет типы и транспилирует `.ts` в `.js` |
